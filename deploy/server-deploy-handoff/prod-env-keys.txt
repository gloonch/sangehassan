# Production env keys

Use one env file per VPS:
- prod-com: `deploy/.prod-com.env`
- prod-ir: `deploy/.prod-ir.env`

Required backend/runtime keys:
- `APP_ENV=production`
- `PORT=8080`
- `DB_HOST=db`
- `DB_PORT=5432`
- `DB_USER=sangehassan`
- `DB_PASSWORD=<strong-secret>`
- `DB_NAME=sangehassan`
- `DB_SSLMODE=disable`
- `JWT_SECRET=<strong-secret>`
- `JWT_TTL_HOURS=24`
- `ACCESS_TOKEN_MINUTES=15`
- `REFRESH_TOKEN_DAYS=30`
- `COOKIE_SECURE=true`
- `ALLOWED_ORIGINS=<comma-separated allowed browser origins>`

Frontend build keys:
- `VITE_API_BASE_URL=` defaults to same-origin `/api`
- `VITE_IMAGE_BASE_URL=` defaults to same-origin `/images`
- `VITE_SITE_URL=https://sangehassan.com` on prod-com
- `VITE_SITE_URL=https://sangehassan.com` on prod-ir too; the `.ir` nginx host redirects to `.com`
- `VITE_PRERENDER_API_BASE_URL=https://sangehassan.com` so dynamic detail pages are prerendered into HTML and sitemap when the public API is reachable during frontend image build
- `VITE_SITEMAP_LASTMOD=` optional; leave blank to use the frontend build date for sitemap `lastmod`

Current target origins:
- prod-com: `https://sangehassan.com,https://www.sangehassan.com,http://sangehassan.com,http://www.sangehassan.com,http://91.239.211.60`
- prod-ir: `.ir` only redirects; keep backend browser origins aligned with the canonical `.com` origin if this stack is still built

Run commands:
- prod-com: `docker compose --env-file .prod-com.env -f docker-compose-prod-com.yml up -d --build`
- prod-ir: `docker compose --env-file .prod-ir.env -f docker-compose-prod-ir.yml up -d --build`
